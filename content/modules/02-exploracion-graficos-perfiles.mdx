## Por qué empezar por la exploración gráfica

Antes de ajustar cualquier modelo, la exploración visual de los datos longitudinales tiene tres objetivos concretos:

1. **Confirmar o descartar supuestos** antes de especificar la estructura aleatoria: ¿las trayectorias son
   aproximadamente paralelas (solo intercepto aleatorio) o divergen (pendiente aleatoria necesaria)?
2. **Detectar anomalías**: sujetos con trayectorias atípicas, mediciones faltantes, valores imposibles.
3. **Formular hipótesis sobre la estructura del modelo**: ¿hay diferencias entre grupos? ¿el efecto del tiempo
   es lineal o no lineal? ¿la variabilidad cambia con el tiempo?

Un buen EDA longitudinal no reemplaza al modelo, pero evita especificaciones incorrectas que ningún criterio de
ajuste posterior puede corregir.

## Datos: crecimiento de abetos de Sitka bajo ozono (SPRUCE1)

El dataset `SPRUCE1` proviene de un experimento clásico de contaminación atmosférica: árboles de Sitka (*Picea
sitchensis*) fueron sometidos a dos condiciones, **Control** (cámara sin contaminante) y **Ozone** (cámara con
ozono elevado), y medidos repetidamente a lo largo del tiempo. La respuesta es el **logaritmo del tamaño**
(`log_growth`), que linealiza el crecimiento inherentemente exponencial de la madera.

Este diseño es el escenario de referencia del **Módulo 3 (Clases 3 y 5)** del curso: dos grupos, medidas
repetidas, heterogeneidad entre árboles dentro de cada grupo.

<Callout type="info" title="Por qué log(tamaño) y no tamaño">
  El crecimiento de los árboles es multiplicativo: un árbol no crece sumando centímetros fijos sino multiplicando
  su tamaño por un factor. Aplicar logaritmo convierte esa relación en aditiva, lo que hace plausible asumir
  errores normales aditivos. Es una transformación de Box-Cox estándar para datos de crecimiento biológico.
</Callout>

**Qué leer en esta tabla:**

- Comparar la columna `media` entre **Control** y **Ozone** en cada tiempo: ¿en qué momento empiezan a divergir?
- Observar si la `sd` de Ozone supera sistemáticamente a la de Control: indicaría que el ozono no solo reduce
  el crecimiento sino que también aumenta la variabilidad entre árboles.
- Si los grupos tienen el mismo `n_arboles` en todos los tiempos, el diseño es balanceado y el análisis
  es más sencillo.

## Gráficos de perfiles

### 1 · Perfiles individuales por grupo (spaghetti)

<RCell id="spruce-spaghetti" title="Spaghetti plot: perfiles individuales coloreados por grupo" code={`library(ggplot2)

ggplot(data, aes(x = time1, y = log_growth,
                 group = ID, color = Group)) +
  geom_line(alpha = 0.45, linewidth = 0.55) +
  geom_point(alpha = 0.45, size = 1.2) +
  scale_colour_manual(values = c("Control" = "steelblue",
                                 "Ozone"   = "firebrick")) +
  scale_x_continuous(breaks = sort(unique(data$time1))) +
  labs(
    title    = "Perfiles de crecimiento individual — Control vs Ozono",
    subtitle = "Cada línea es un árbol; el color indica el grupo experimental",
    x        = "Tiempo (días desde inicio del experimento)",
    y        = "log(tamaño del árbol)",
    color    = "Grupo",
    caption  = "Fuente: SPRUCE1 — Clases 3 y 5, J. C. Salazar"
  ) +
  theme_minimal(base_size = 12) +
  theme(
    panel.grid.minor    = element_blank(),
    legend.position     = "bottom"
  )`} />

**Qué observar:**

- ¿Las líneas de los dos grupos se separan claramente o se superponen? Una separación creciente con el tiempo
  indica una **interacción grupo × tiempo**.
- ¿Dentro de cada grupo las líneas son aproximadamente paralelas (solo intercepto aleatorio necesario) o
  divergen (pendiente aleatoria necesaria)?
- ¿Hay árboles con trayectorias anómalas (caída brusca, salto sin sentido)? Pueden indicar error de medición.

### 2 · Rejilla por grupo: cada árbol en su panel

Ver cada árbol individualmente, organizados en una rejilla y separados por grupo, revela la heterogeneidad
intra-grupo de forma mucho más clara que el spaghetti plot.

<RCell id="spruce-facet-id" title="Rejilla de perfiles individuales — organizada por grupo" code={`library(ggplot2)

ggplot(data, aes(x = time1, y = log_growth)) +
  geom_line(aes(color = Group), linewidth = 0.65) +
  geom_point(aes(color = Group), size = 1.3) +
  geom_smooth(method  = "lm", se = FALSE,
              color   = "gray30", linewidth = 0.5,
              linetype = "dashed") +
  facet_wrap(~ ID, ncol = 8) +
  scale_colour_manual(values = c("Control" = "steelblue",
                                 "Ozone"   = "firebrick")) +
  scale_x_continuous(breaks = sort(unique(data$time1)),
                     labels = function(x) format(x, big.mark = "")) +
  labs(
    title    = "Trayectoria de cada árbol con tendencia lineal (gris discontinua)",
    subtitle = "La heterogeneidad de interceptos y pendientes dentro de cada grupo motiva los efectos aleatorios",
    x        = "Tiempo",
    y        = "log(crecimiento)",
    color    = "Grupo"
  ) +
  theme_minimal(base_size = 8) +
  theme(
    strip.text       = element_text(size = 7, face = "bold"),
    panel.grid.minor = element_blank(),
    legend.position  = "bottom",
    axis.text.x      = element_text(angle = 45, hjust = 1, size = 6)
  )`} />

**Qué buscar:**

- Paneles donde la línea gris (tendencia lineal) es una mala representación de los puntos: indicios de
  no-linealidad que el modelo puede necesitar capturar con splines o polinomios.
- Árboles con pocos puntos (mediciones faltantes): si son muchos, considerar si el dato faltante es aleatorio
  o sistemático (un árbol que murió no es faltante aleatorio).

### 3 · Perfil promedio por grupo con banda de variabilidad

El perfil promedio resume la **tendencia poblacional de cada grupo**. Agregarle una banda de ±1 SD o el
intervalo de error estándar comunica cuánta variabilidad hay alrededor de esa tendencia.

<RCell id="spruce-mean-ribbon" title="Perfil promedio ± SD por grupo" code={`library(ggplot2)
library(dplyr)

# Calcular media y SD por grupo y tiempo
resumen <- data |>
  group_by(Group, time1) |>
  summarise(
    media = mean(log_growth, na.rm = TRUE),
    sd    = sd(log_growth,   na.rm = TRUE),
    n     = n(),
    se    = sd / sqrt(n),
    .groups = "drop"
  )

ggplot(resumen, aes(x = time1, y = media,
                    color = Group, fill = Group)) +
  # Banda ± 1 SD (variabilidad entre árboles)
  geom_ribbon(aes(ymin = media - sd, ymax = media + sd),
              alpha = 0.15, color = NA) +
  # Banda ± 1 SE (incertidumbre del promedio)
  geom_ribbon(aes(ymin = media - se, ymax = media + se),
              alpha = 0.35, color = NA) +
  geom_line(linewidth = 1.3) +
  geom_point(size = 3) +
  scale_colour_manual(values = c("Control" = "steelblue",
                                 "Ozone"   = "firebrick")) +
  scale_fill_manual(values   = c("Control" = "steelblue",
                                 "Ozone"   = "firebrick")) +
  scale_x_continuous(breaks = sort(unique(data$time1))) +
  labs(
    title    = "Perfil promedio de log(crecimiento) por grupo",
    subtitle = "Banda oscura = ±1 SE (incertidumbre del promedio)  ·  Banda clara = ±1 SD (variabilidad entre árboles)",
    x        = "Tiempo",
    y        = "log(crecimiento) promedio",
    color    = "Grupo", fill = "Grupo"
  ) +
  theme_minimal(base_size = 12) +
  theme(
    panel.grid.minor = element_blank(),
    legend.position  = "bottom"
  )`} />

**Cómo leer este gráfico:**

- La **separación vertical** entre las dos líneas en cada tiempo es la estimación descriptiva del efecto del
  ozono en ese momento. Si las bandas de SE no se solapan, hay evidencia exploratoria de diferencia.
- El **tamaño de la banda de SD** refleja la heterogeneidad entre árboles dentro de cada grupo: si la banda
  de Ozone es más ancha, el tratamiento introduce más variabilidad individual.
- Una **divergencia progresiva** entre las curvas indica interacción grupo × tiempo, lo que en el modelo se
  representa como un coeficiente de interacción.

### 4 · Perfiles individuales + promedio superpuestos

Combinar ambas capas en un mismo gráfico permite juzgar qué tan bien representa el promedio la distribución
de trayectorias reales.

<RCell id="spruce-combined" title="Perfiles individuales y promedio en un panel" code={`library(ggplot2)
library(dplyr)

resumen <- data |>
  group_by(Group, time1) |>
  summarise(media = mean(log_growth, na.rm = TRUE), .groups = "drop")

ggplot() +
  # Capa 1: trayectorias individuales (fondo, semitransparente)
  geom_line(data  = data,
            aes(x = time1, y = log_growth,
                group = ID, color = Group),
            alpha = 0.25, linewidth = 0.45) +
  # Capa 2: perfil promedio (primer plano, línea gruesa)
  geom_line(data  = resumen,
            aes(x = time1, y = media,
                group = Group, color = Group),
            linewidth = 1.6) +
  geom_point(data = resumen,
             aes(x = time1, y = media,
                 color = Group),
             size = 3.5, shape = 21,
             fill = "white", stroke = 1.5) +
  scale_colour_manual(values = c("Control" = "steelblue",
                                 "Ozone"   = "firebrick")) +
  scale_x_continuous(breaks = sort(unique(data$time1))) +
  labs(
    title    = "Trayectorias individuales y perfil promedio por grupo",
    subtitle = "Líneas tenues = árboles individuales · Línea gruesa = media del grupo",
    x        = "Tiempo",
    y        = "log(crecimiento)",
    color    = "Grupo"
  ) +
  theme_minimal(base_size = 12) +
  theme(
    panel.grid.minor = element_blank(),
    legend.position  = "bottom"
  )`} />

**Qué leer:**

- Si las líneas individuales están muy dispersas alrededor del promedio, la **varianza del intercepto aleatorio**
  será grande en el modelo.
- Si las trayectorias de Ozone convergen hacia el promedio más que las de Control (o viceversa), puede indicar
  que el tratamiento reduce la heterogeneidad.

### 5 · Distribución en cada tiempo: boxplot temporal por grupo

Este gráfico organiza la información por **tiempo de medición** (eje X), mostrando la distribución completa
de `log_growth` en cada ocasión, separada por grupo.

<RCell id="spruce-temporal-boxplot" title="Boxplot temporal — distribución en cada ocasión de medición" code={`library(ggplot2)

ggplot(data, aes(x = factor(time1), y = log_growth,
                 fill = Group)) +
  geom_boxplot(alpha  = 0.55,
               outlier.shape = 21,
               outlier.size  = 1.5,
               position      = position_dodge(0.8),
               width         = 0.65) +
  scale_fill_manual(values = c("Control" = "steelblue",
                               "Ozone"   = "firebrick")) +
  labs(
    title    = "Distribución de log(crecimiento) en cada tiempo, por grupo",
    subtitle = "Cada par de cajas compara Control (azul) vs Ozone (rojo) en la misma ocasión",
    x        = "Tiempo de medición",
    y        = "log(crecimiento)",
    fill     = "Grupo"
  ) +
  theme_minimal(base_size = 12) +
  theme(
    panel.grid.minor = element_blank(),
    legend.position  = "bottom"
  )`} />

**Cómo leer el boxplot temporal:**

- **Separación entre cajas** en cada tiempo: diferencia descriptiva entre grupos. Si la separación crece con
  el tiempo, la interacción grupo × tiempo es evidente.
- **Tamaño de las cajas** (IQR): variabilidad entre árboles en ese tiempo. Un IQR creciente con el tiempo
  indica heterocedasticidad o divergencia de trayectorias.
- **Outliers** (puntos fuera del bigote): árboles con crecimiento inusualmente alto o bajo en ese momento.
  Si el mismo árbol aparece como outlier en múltiples tiempos, puede tener un intercepto aleatorio extremo.

### 6 · Comparación global: densidades por grupo

Este gráfico **ignora el tiempo** (colapsa todas las mediciones) y sirve únicamente como resumen marginal.
Es útil para comunicar la diferencia entre grupos de forma simple, pero no debe usarse como única evidencia.

<RCell id="spruce-density" title="Densidad global de log(crecimiento) por grupo" code={`library(ggplot2)

ggplot(data, aes(x = log_growth, fill = Group, color = Group)) +
  geom_density(alpha = 0.35, linewidth = 0.9) +
  geom_vline(data = data |>
               group_by(Group) |>
               summarise(media = mean(log_growth, na.rm = TRUE)),
             aes(xintercept = media, color = Group),
             linetype = "dashed", linewidth = 0.9) +
  scale_fill_manual(values  = c("Control" = "steelblue",
                                "Ozone"   = "firebrick")) +
  scale_colour_manual(values = c("Control" = "steelblue",
                                 "Ozone"   = "firebrick")) +
  labs(
    title    = "Distribución marginal de log(crecimiento) por grupo",
    subtitle = "Línea discontinua = media del grupo  ·  Nota: ignora la estructura temporal",
    x        = "log(crecimiento)",
    y        = "Densidad",
    fill     = "Grupo", color = "Grupo"
  ) +
  theme_minimal(base_size = 12) +
  theme(
    panel.grid.minor = element_blank(),
    legend.position  = "bottom"
  )`} />

<Callout type="warning" title="Limitación de la comparación marginal">
  Esta densidad colapsa todas las mediciones repetidas de un mismo árbol como si fueran independientes.
  Sobrestima el tamaño de muestra efectivo y no captura la correlación intra-árbol. Es útil como presentación
  visual preliminar, no como base para un test estadístico.
</Callout>

## Preguntas que debería responder el EDA antes de modelar

Una exploración bien hecha lleva a respuestas tentativas para estas preguntas:

1. ¿El efecto del tiempo es **lineal** o necesita una función más flexible?
2. ¿Hay una diferencia entre grupos que **crece con el tiempo** (interacción) o es constante (efecto principal)?
3. ¿La variabilidad entre árboles parece mayor en un grupo que en otro (¿varianzas heterogéneas entre grupos)?
4. ¿El modelo de partida razonable es un LMM con **intercepto aleatorio solamente**, o necesitamos también
   **pendiente aleatoria**?
5. ¿Hay árboles que deberían examinarse con más cuidado antes de incluirlos en el análisis?

Llevar estas respuestas tentativas a la especificación del modelo es lo que se trabaja en los módulos 3 y 4.

  SERIES x=time1 y=log_growth / GROUP=ID GROUPLC=Group
         LINEATTRS=(THICKNESS=1) TRANSPARENCY=0.6;
  XAXIS LABEL="Tiempo" VALUES=(...);
  YAXIS LABEL="log(crecimiento)";
  KEYLEGEND / TITLE="";
RUN;

/* Perfil promedio ± SE por grupo */
PROC SGPLOT DATA=spruce1;
  BAND    x=time1 LOWER=media_menos_se UPPER=media_mas_se
          / GROUP=Group TRANSPARENCY=0.6;
  VLINE   time1 / RESPONSE=log_growth GROUP=Group STAT=MEAN
                  LIMITSTAT=CLM MARKERS LINEATTRS=(THICKNESS=2);
  XAXIS   LABEL="Tiempo";
  YAXIS   LABEL="log(crecimiento) promedio ± SE";
RUN;

/* Boxplot temporal por grupo (usando SGPANEL para facets) */
PROC SGPANEL DATA=spruce1;
  PANELBY Group / ROWS=2 UNISCALE=ALL;
  VBOX log_growth / CATEGORY=time1
                    FILLATTRS=(TRANSPARENCY=0.4);
  COLAXIS LABEL="Tiempo de medición";
  ROWAXIS LABEL="log(crecimiento)";
RUN;`} />
