## Del análisis exploratorio a la especificación del modelo

El EDA del Módulo 1 mostró dos señales claras en `sleepstudy`:

1. **Los interceptos varían entre sujetos**: en el día 0, el tiempo de reacción oscila de ~200 ms a ~310 ms.
2. **Las pendientes también varían**: algunos sujetos se deterioran ≈5 ms/día, otros ≈25 ms/día.

Esas dos observaciones empíricas se traducen directamente en decisiones de especificación del modelo. Esta
sección formaliza ese razonamiento.

---

## Las tres variantes fundamentales de la estructura aleatoria

Un LMM con respuesta $y_{ij}$ (sujeto $i$, tiempo $j$) y un predictor continuo $t_{ij}$ puede tener tres
configuraciones comunes de efectos aleatorios. Cada una responde a una pregunta diferente sobre la heterogeneidad
de la población.

<DataTable

  columns={["Variante", "Fórmula aleatoria en R", "Parámetros libres en D", "Cuándo usarla"]}
  rows={[
    [
      "A — Solo intercepto",
      "(1 | Subject)",
      "1 (τ₀²)",
      "Perfiles paralelos en EDA; solo varía el nivel basal"
    ],
    [
      "B — Intercepto + pendiente (correlación libre)",
      "(time | Subject)",
      "3 (τ₀², τ₁², τ₀₁)",
      "Perfiles que divergen o se cruzan; sujetos con tasas de cambio distintas"
    ],
    [
      "C — Intercepto + pendiente (correlación = 0)",
      "(1 | Subject) + (0 + time | Subject)",
      "2 (τ₀², τ₁²)",
      "Como B, pero pocos datos por sujeto o problemas de convergencia"
    ]
  ]}
/>

<Callout type="info" title="La notación (time | Subject) es abreviatura de (1 + time | Subject)">
  El intercepto aleatorio siempre está implícito cuando la fórmula tiene más de un término a la izquierda de
  `|`. Para excluir el intercepto aleatorio (raramente justificado) se escribe `(0 + time | Subject)`.
</Callout>

## Variante A: solo intercepto aleatorio

### Qué dice conceptualmente

Este modelo asume que **todos los sujetos tienen la misma pendiente** (la misma tasa de cambio con el tiempo),
pero cada uno parte de un **nivel basal distinto**. La heterogeneidad entre sujetos es solo de "posición":
los perfiles individuales son líneas paralelas, desplazadas verticalmente.

$$
y_{ij} = (\beta_0 + b_{0i}) + \beta_1\, t_{ij} + \varepsilon_{ij}, \qquad b_{0i} \sim N(0, \tau_0^2)
$$

La varianza total de $y_{ij}$ se descompone en dos fuentes:
- $\tau_0^2$: varianza **entre sujetos** (heterogeneidad de niveles basales).
- $\sigma^2$: varianza **intra-sujeto** (ruido residual).

El **coeficiente de correlación intraclase** (ICC) resume cuánta fracción de la variabilidad total es entre
sujetos:

$$
\text{ICC} = \frac{\tau_0^2}{\tau_0^2 + \sigma^2}
$$

Un ICC alto (> 0.5) indica que la mayor parte de la variación está entre sujetos, justificando el modelo
mixto frente a la regresión clásica.

### Cuándo está justificada

- La exploración gráfica muestra perfiles aproximadamente **paralelos** (las líneas del spaghetti plot tienen
  pendientes similares).
- La teoría no sugiere que las unidades deberían tener tasas de cambio distintas (ej: el efecto del tiempo es
  el mismo para todos los pacientes de un grupo homogéneo).

### Código R y SAS

<RCell id="mod-ri" title="Variante A: solo intercepto aleatorio" code={`library(lme4)
data(sleepstudy)

# Variante A: solo intercepto aleatorio
mod_A <- lmer(Reaction ~ Days + (1 | Subject),
              data    = sleepstudy,
              REML    = TRUE)

summary(mod_A)

# ICC: fracción de varianza entre sujetos
vc <- as.data.frame(VarCorr(mod_A))
tau0_sq <- vc[vc$grp == "Subject", "vcov"]
sigma_sq <- vc[vc$grp == "Residual", "vcov"]

cat("ICC =", round(tau0_sq / (tau0_sq + sigma_sq), 3), "\n")`} />

<SASAccordion title="Variante A en SAS PROC MIXED" code={`/* Variante A: solo intercepto aleatorio */
PROC MIXED DATA=sleepstudy METHOD=REML COVTEST;
  CLASS Subject;
  MODEL Reaction = Days / SOLUTION DDFM=KENWARDROGER;
  RANDOM INTERCEPT / SUBJECT=Subject TYPE=VC;
  /* TYPE=VC: varianza constante — un solo parámetro tau0^2 */
RUN;

/* El output incluye:
   - "Covariance Parameter Estimates": tau0^2 y sigma^2
   - "Solution for Fixed Effects": beta0 y beta1 con sus SE
   ICC = tau0^2 / (tau0^2 + sigma^2) se calcula manualmente */`} />

## Variante B: intercepto y pendiente aleatorios (correlación libre)

### Qué añade conceptualmente

Este modelo reconoce que las unidades **difieren tanto en su nivel basal como en su tasa de cambio**. Además,
permite que ambas características estén relacionadas: por ejemplo, en `sleepstudy`, ¿los sujetos con peor
tiempo de reacción basal tienden también a deteriorarse más rápido?

$$
y_{ij} = (\beta_0 + b_{0i}) + (\beta_1 + b_{1i})\, t_{ij} + \varepsilon_{ij}
$$

$$
\begin{pmatrix} b_{0i} \\ b_{1i} \end{pmatrix} \sim N\!\left(
  \begin{pmatrix}0\\0\end{pmatrix},
  \begin{pmatrix}\tau_0^2 & \tau_{01} \\ \tau_{01} & \tau_1^2\end{pmatrix}
\right)
$$

La **correlación entre efectos aleatorios** $\rho_{01} = \tau_{01}/(\tau_0\,\tau_1)$ tiene una interpretación
sustantiva: un $\rho_{01}$ positivo significa que los sujetos que parten más alto también crecen más rápido
(o se deterioran más); un $\rho_{01}$ negativo indica convergencia (los que parten más alto cambian menos).

### Cuándo está justificada

- El spaghetti plot o la rejilla de perfiles muestran **trayectorias que divergen o se cruzan**.
- El EDA mostró que la correlación individual Days→Reaction varía entre sujetos (Módulo 1).
- Hay suficientes mediciones por sujeto para estimar dos parámetros aleatorios (regla práctica: ≥5 mediciones
  por unidad).
- La teoría sugiere que el punto de partida y la velocidad de cambio están relacionados.

### Código R y SAS

<RCell id="mod-ri-rs" title="Variante B: intercepto y pendiente aleatorios (D sin restricciones)" code={`library(lme4)
data(sleepstudy)

# Variante B: intercepto y pendiente aleatorios, correlación libre
# (1 + Days | Subject) equivale a (Days | Subject)
mod_B <- lmer(Reaction ~ Days + (Days | Subject),
              data = sleepstudy,
              REML = TRUE)

summary(mod_B)

# La sección "Random effects" muestra:
# - Std.Dev (Subject, Intercept): sqrt(tau0^2)  — heterogeneidad de niveles basales
# - Std.Dev (Subject, Days):      sqrt(tau1^2)  — heterogeneidad de pendientes
# - Corr:                         rho_01         — ¿se relacionan intercept y pendiente?
# - Std.Dev (Residual):           sigma          — ruido residual intra-sujeto

# Matriz D estimada
VarCorr(mod_B)`} />

<SASAccordion title="Variante B en SAS PROC MIXED" code={`/* Variante B: intercepto y pendiente aleatorios, D sin restricciones (UN) */
PROC MIXED DATA=sleepstudy METHOD=REML COVTEST;
  CLASS Subject;
  MODEL Reaction = Days / SOLUTION DDFM=KENWARDROGER;
  RANDOM INTERCEPT Days / SUBJECT=Subject TYPE=UN G GCORR;
  /* TYPE=UN: matriz D sin restricciones — estima tau0^2, tau1^2 y tau01
     G: imprime la matriz D estimada
     GCORR: imprime la correlación rho01 entre intercepto y pendiente */
RUN;`} />

## Variante C: intercepto y pendiente aleatorios (correlación forzada a 0)

### Por qué forzar la correlación a cero

Estimar la correlación $\rho_{01}$ consume un parámetro adicional y puede generar inestabilidad numérica
cuando hay pocos sujetos o pocas mediciones por sujeto. Forzar $\tau_{01} = 0$ simplifica la estructura D:

$$
\mathbf{D} = \begin{pmatrix}\tau_0^2 & 0 \\ 0 & \tau_1^2\end{pmatrix}
$$

Esto no niega que los efectos aleatorios existan; solo asume que el nivel basal y la tasa de cambio de un
sujeto no están asociados en la población.

<Callout type="warning" title="Cuándo NO usar la Variante C">
  Si la teoría o el EDA sugieren una relación entre el nivel basal y la pendiente (por ejemplo, sujetos más
  deteriorados se recuperan menos), forzar $\tau_{01} = 0$ produce un modelo mal especificado. Es preferible
  usar la Variante B y lidiar con la inestabilidad numérica cambiando el optimizador.
</Callout>

### Código R y SAS

<RCell id="mod-ri-rs-indep" title="Variante C: intercepto y pendiente sin correlación" code={`library(lme4)
data(sleepstudy)

# Variante C: dos efectos aleatorios INDEPENDIENTES
# (1 | Subject) + (0 + Days | Subject) fuerza tau01 = 0
mod_C <- lmer(Reaction ~ Days + (1 | Subject) + (0 + Days | Subject),
              data = sleepstudy,
              REML = TRUE)

summary(mod_C)
VarCorr(mod_C)

# Notar: ya no aparece "Corr" en la sección de Random effects`} />

<SASAccordion title="Variante C en SAS (TYPE=VC para ambas componentes)" code={`/* Variante C: intercepto y pendiente aleatorios, covarianza = 0 */
PROC MIXED DATA=sleepstudy METHOD=REML COVTEST;
  CLASS Subject;
  MODEL Reaction = Days / SOLUTION DDFM=KENWARDROGER;
  RANDOM INTERCEPT / SUBJECT=Subject TYPE=VC;
  RANDOM Days      / SUBJECT=Subject TYPE=VC;
  /* Dos sentencias RANDOM separadas con TYPE=VC fuerzan
     la covarianza entre intercepto y pendiente a cero */
RUN;`} />

## Cómo elegir entre variantes: criterios de datos y teoría

La elección no es solo estadística. Combine siempre los dos planos:

<ProcessSteps steps={[
  {
    title: "Plano teórico: pregúntese si la heterogeneidad tiene sentido",
    description: "¿Es razonable que cada sujeto tenga su propia tasa de cambio? Si estás midiendo el efecto de una droga en pacientes con la misma condición basal, quizás la pendiente aleatoria no está justificada por la teoría aunque los datos la sugieran."
  },
  {
    title: "Plano empírico: lea el spaghetti plot y la correlación por sujeto",
    description: "Perfiles paralelos → solo intercepto aleatorio (Var. A). Perfiles que divergen o se cruzan → considere pendiente aleatoria (Var. B o C). La correlación por sujeto del Módulo 1 cuantifica esta heterogeneidad."
  },
  {
    title: "Plano estadístico: compare modelos anidados",
    description: "Use anova(mod_A, mod_B, refit=FALSE) para un LRT entre variantes. Complementariamente, compare AIC y BIC. Si el modelo más complejo no mejora significativamente, quédese con el más simple (parsimonia)."
  },
  {
    title: "Plano numérico: verifique convergencia",
    description: "Si lmer() da advertencia de singular fit o de convergencia fallida con la Var. B, intente la Var. C primero (menos parámetros), luego cambie el optimizador a bobyqa, y finalmente escale los predictores."
  }
]} />

### Comparación estadística entre variantes

<RCell id="compare-variants" title="Comparar Variante A vs B con LRT y AIC/BIC" code={`library(lme4)
data(sleepstudy)

mod_A <- lmer(Reaction ~ Days + (1 | Subject),          data = sleepstudy, REML = TRUE)
mod_B <- lmer(Reaction ~ Days + (Days | Subject),        data = sleepstudy, REML = TRUE)
mod_C <- lmer(Reaction ~ Days + (1 | Subject)
                            + (0 + Days | Subject),      data = sleepstudy, REML = TRUE)

# LRT entre modelos anidados (refit=FALSE mantiene REML para comparar SOLO la estructura aleatoria)
anova(mod_A, mod_B, refit = FALSE)

# Tabla de AIC / BIC para los tres
AIC(mod_A, mod_B, mod_C)
BIC(mod_A, mod_B, mod_C)`} />

**Cómo leer la comparación:**

- El **LRT** (estadístico $\chi^2$ y valor p en `anova()`) contrasta si el modelo más complejo ajusta
  significativamente mejor. Un $p < 0.05$ favorece la variante más compleja.
- El **AIC** penaliza la verosimilitud por el número de parámetros; el modelo con menor AIC es preferible
  cuando hay incertidumbre sobre la estructura correcta.
- El **BIC** penaliza más fuertemente la complejidad; suele favorecer modelos más simples que el AIC.
- Si LRT y AIC apuntan al modelo más complejo pero BIC al más simple, deberá apoyarse en la **teoría**
  para decidir.

<Callout type="warning" title="LRT con REML solo para comparar estructura aleatoria">
  `anova(mod_A, mod_B, refit=FALSE)` compara la estructura **aleatoria** mientras mantiene los mismos efectos
  fijos. Para comparar **efectos fijos**, debe usar `REML=FALSE` en ambos modelos. Ver Módulo 8.
</Callout>

## Guía completa de lectura del output de summary(lmer())

<RCell id="output-guide" title="Output completo del modelo B con sleepstudy" code={`library(lme4)
data(sleepstudy)

mod_B <- lmer(Reaction ~ Days + (Days | Subject),
              data = sleepstudy, REML = TRUE)
summary(mod_B)`} />

La salida tiene cinco bloques. Aquí se explica cada uno:

### Bloque 1 — Fórmula y método de estimación

```
Linear mixed model fit by REML ['lmerMod']
Formula: Reaction ~ Days + (Days | Subject)
   Data: sleepstudy
REML criterion at convergence: 1743.6
```

- **REML criterion**: la deviance REML (−2 × log-verosimilitud restringida). Solo comparable entre modelos con
  los mismos efectos fijos.

### Bloque 2 — Residuos escalados

```
Scaled residuals:
    Min      1Q  Median      3Q     Max
-3.9536 -0.4634  0.0231  0.4634  5.1793
```

- Si la distribución es aproximadamente normal, los cuartiles deberían estar cerca de ±0.67 y el mínimo/máximo
  dentro de ±3.5. Valores extremos indican posibles outliers.

### Bloque 3 — Efectos aleatorios ← **el corazón del LMM**

```
Random effects:
 Groups   Name        Variance Std.Dev. Corr
 Subject  (Intercept) 612.10   24.741
          Days         35.07    5.922   0.07
 Residual             654.94   25.592
Number of obs: 180, groups:  Subject, 18
```

| Componente | Notación | Qué mide |
|---|---|---|
| Variance (Intercept, Subject) | $\tau_0^2 = 612$ | Cuánto varían los tiempos de reacción basales entre sujetos |
| Variance (Days, Subject) | $\tau_1^2 = 35$ | Cuánto varían las pendientes (tasas de deterioro) entre sujetos |
| Corr | $\rho_{01} = 0.07$ | ¿Los que parten más alto se deterioran más rápido? (correlación casi nula aquí) |
| Residual | $\sigma^2 = 655$ | Variabilidad intra-sujeto no explicada por el modelo |

La **desviación estándar** (Std.Dev.) es la raíz cuadrada de la varianza y está en las mismas unidades que
$y$ (ms). Mide la dispersión de los efectos aleatorios en la población.

### Bloque 4 — Efectos fijos

```
Fixed effects:
            Estimate Std. Error t value
(Intercept)  251.405      6.825  36.84
Days          10.467      1.546   6.77
```

- **Intercept (251.4 ms)**: tiempo de reacción promedio poblacional en el día 0.
- **Days (10.5 ms/día)**: por cada día adicional de privación, el tiempo de reacción promedio aumenta 10.5 ms.
- **Std. Error**: incertidumbre del estimador puntual. El `lme4` no reporta p-valores por defecto; para
  obtenerlos use `lmerTest::lmer()` o el paquete `lmerTest`.

### Bloque 5 — Correlación de los efectos fijos

```
Correlation of Fixed Effects:
     (Intr)
Days -0.138
```

- Esta es la **correlación entre los estimadores** $\hat\beta_0$ y $\hat\beta_1$, no entre los efectos
  aleatorios. Un valor cercano a 0 (como aquí) indica que las estimaciones son casi ortogonales.
- Una correlación alta (> ±0.7) sugiere que centrar el predictor de tiempo mejoraría la interpretación.

## Valores ajustados y residuales

<RCell id="fitted-residuals" title="Extraer fitted values, residuales y efectos aleatorios" code={`library(lme4)
library(ggplot2)
data(sleepstudy)

mod_B <- lmer(Reaction ~ Days + (Days | Subject),
              data = sleepstudy, REML = TRUE)

# Valores ajustados (incluyen el efecto aleatorio de cada sujeto)
sleepstudy$fitted   <- fitted(mod_B)

# Residuales (diferencia entre observado y ajustado)
sleepstudy$resid    <- resid(mod_B)

# Efectos aleatorios estimados (BLUPs): b0i y b1i para cada sujeto
ranef(mod_B)$Subject

# Visualización: ajuste por sujeto
ggplot(sleepstudy, aes(x = Days)) +
  geom_point(aes(y = Reaction), alpha = 0.4, size = 1.5) +
  geom_line(aes(y = fitted), color = "steelblue", linewidth = 1) +
  facet_wrap(~ Subject, ncol = 6) +
  scale_x_continuous(breaks = c(0, 3, 6, 9)) +
  labs(
    title    = "Rectas ajustadas del LMM por sujeto (Variante B)",
    subtitle = "Cada recta usa el intercepto y la pendiente específicos del sujeto (β + b_i)",
    x = "Días", y = "Tiempo de reacción (ms)"
  ) +
  theme_minimal(base_size = 9) +
  theme(strip.text = element_text(size = 8, face = "bold"),
        panel.grid.minor = element_blank())`} />

**Qué observar:**

- Las rectas **no son paralelas**: cada sujeto tiene su propio intercepto y su propia pendiente (eso es la
  Variante B).
- Los sujetos cuya recta sube poco son los "resistentes" a la privación de sueño; los de recta empinada, los
  más vulnerables.
- Compare este gráfico con el de la Variante A (`(1 | Subject)`, rectas paralelas) para ver visualmente
  lo que aporta la pendiente aleatoria.
