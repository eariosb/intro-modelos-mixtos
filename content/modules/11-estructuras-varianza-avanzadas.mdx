## ¿Por qué importa la estructura de covarianza?

En estudios longitudinales, las observaciones de un mismo sujeto están correlacionadas. Ignorar esa correlación no siempre sesga los coeficientes estimados, pero sí afecta la **eficiencia estadística**: los errores estándar pueden ser incorrectos y se pierde potencia para detectar efectos relevantes.

En `nlme` disponemos de dos herramientas complementarias:
- **Modelos mixtos (`lme`)**: los efectos aleatorios inducen una estructura de correlación; las funciones `cor*` permiten refinarla.
- **Modelos marginales (`gls`)**: no incluyen efectos aleatorios, pero modelan directamente la correlación y la varianza.

Ambos enfoques son útiles y comparten las mismas funciones `cor*` y `var*`. En este módulo exploramos cuándo y cómo elegir entre las opciones disponibles.

<Callout type="info" title="Modelos condicionales vs. marginales en nlme">
  - `lme(random = ~ 1 | Subject, correlation = corAR1())` → modelo **condicional** con intercepto aleatorio más correlación AR(1).  
  - `gls(correlation = corAR1())` → modelo **marginal** basado en verosimilitud, que modela la correlación sin efectos aleatorios.  
  Ambos son válidos; la elección depende de si necesitás los BLUPs de los efectos aleatorios o solo la estimación de los efectos fijos poblacionales.
</Callout>

## 2. Comparación de estructuras de correlación

Ajustamos varios modelos con `lme`, variando únicamente la estructura de correlación, y comparamos criterios de información. Usamos `method = "ML"` para que la comparación por AIC/BIC sea válida.

<RCell id="comparar-correlaciones" title="Ajuste de cuatro estructuras de correlación" code={`library(nlme)

# Simetría compuesta (correlación constante entre cualquier par de mediciones)
mod_cs <- lme(distance ~ age + Sex, random = ~ 1 | Subject,
              correlation = corCompSymm(form = ~ 1 | Subject),
              data = Orthodont, method = "ML")

# AR(1): la correlación decae exponencialmente con la distancia temporal
mod_ar1 <- lme(distance ~ age + Sex, random = ~ 1 | Subject,
               correlation = corAR1(form = ~ age | Subject),
               data = Orthodont, method = "ML")

# No estructurada: cada par de tiempos tiene su propia correlación
mod_un <- lme(distance ~ age + Sex, random = ~ 1 | Subject,
              correlation = corSymm(form = ~ 1 | Subject),
              data = Orthodont, method = "ML")

# Exponencial: decaimiento continuo con la distancia temporal
mod_exp <- lme(distance ~ age + Sex, random = ~ 1 | Subject,
               correlation = corExp(form = ~ age | Subject, metric = "euclidean"),
               data = Orthodont, method = "ML")
`} />

<RCell id="tabla-comparativa" title="Comparación de AIC y BIC" code={`library(nlme)

# Recreamos los modelos para asegurar que están en el entorno
mod_cs <- lme(distance ~ age + Sex, random = ~ 1 | Subject,
              correlation = corCompSymm(form = ~ 1 | Subject),
              data = Orthodont, method = "ML")
mod_ar1 <- lme(distance ~ age + Sex, random = ~ 1 | Subject,
               correlation = corAR1(form = ~ age | Subject),
               data = Orthodont, method = "ML")
mod_un <- lme(distance ~ age + Sex, random = ~ 1 | Subject,
              correlation = corSymm(form = ~ 1 | Subject),
              data = Orthodont, method = "ML")
mod_exp <- lme(distance ~ age + Sex, random = ~ 1 | Subject,
               correlation = corExp(form = ~ age | Subject, metric = "euclidean"),
               data = Orthodont, method = "ML")

# Tabla comparativa
comparacion <- data.frame(
  Modelo = c("Simetría compuesta", "AR(1)", "No estructurada", "Exponencial"),
  AIC = round(c(AIC(mod_cs), AIC(mod_ar1), AIC(mod_un), AIC(mod_exp)), 2),
  BIC = round(c(BIC(mod_cs), BIC(mod_ar1), BIC(mod_un), BIC(mod_exp)), 2),
  logLik = round(c(logLik(mod_cs), logLik(mod_ar1), logLik(mod_un), logLik(mod_exp)), 2)
)

print(comparacion)
`} />

<Callout type="tip" title="¿Cómo elegir?">
  - El **menor AIC/BIC** sugiere el mejor equilibrio entre ajuste y complejidad.  
  - La **no estructurada** es la más flexible pero con muchos tiempos se vuelve inestable (muchos parámetros que estimar).  
  - **AR(1)** y **exponencial** asumen que la correlación decae con la distancia temporal: muy razonable en estudios longitudinales.  
  - La **simetría compuesta** asume correlación constante entre cualquier par de mediciones; es simple y útil cuando no hay razón teórica para esperar decaimiento.
</Callout>

## 3. Visualización de las matrices de correlación estimadas

Extraemos las matrices de cada modelo y las visualizamos en un heatmap. Usamos `as.data.frame.table()` en lugar de `reshape2::melt` para mantener la compatibilidad con WebR.

<RCell id="heatmap-matrices" title="Heatmap de las matrices de correlación" code={`library(ggplot2)
library(nlme)

# Ajustamos los modelos con REML (más apropiado para estimación final)
mod_cs <- lme(distance ~ age + Sex, random = ~ 1 | Subject,
              correlation = corCompSymm(form = ~ 1 | Subject),
              data = Orthodont, method = "REML")
mod_ar1 <- lme(distance ~ age + Sex, random = ~ 1 | Subject,
               correlation = corAR1(form = ~ age | Subject),
               data = Orthodont, method = "REML")
mod_un <- lme(distance ~ age + Sex, random = ~ 1 | Subject,
              correlation = corSymm(form = ~ 1 | Subject),
              data = Orthodont, method = "REML")
mod_exp <- lme(distance ~ age + Sex, random = ~ 1 | Subject,
               correlation = corExp(form = ~ age | Subject, metric = "euclidean"),
               data = Orthodont, method = "REML")

# Extraemos las matrices
mat_cs <- corMatrix(mod_cs$modelStruct$corStruct)[[1]]
mat_ar1 <- corMatrix(mod_ar1$modelStruct$corStruct)[[1]]
mat_un <- corMatrix(mod_un$modelStruct$corStruct)[[1]]
mat_exp <- corMatrix(mod_exp$modelStruct$corStruct)[[1]]

# Convertimos a formato largo con función base
mat_to_long <- function(mat, name) {
  df <- as.data.frame.table(mat, responseName = "correlation")

df_all <- rbind(
  mat_to_long(mat_cs, "Simetría compuesta"),
  mat_to_long(mat_ar1, "AR(1)"),
  mat_to_long(mat_un, "No estructurada"),
  mat_to_long(mat_exp, "Exponencial")
)

ggplot(df_all, aes(x = Var1, y = Var2, fill = correlation)) +
  geom_tile(color = "white", linewidth = 0.5) +
  geom_text(aes(label = correlation), size = 3) +
  scale_fill_gradient2(low = "steelblue", mid = "white", high = "coral",
                       midpoint = 0.5, limits = c(0, 1)) +
  facet_wrap(~ Modelo, nrow = 2) +
  labs(title = "Matrices de correlación estimadas",
       subtitle = "Cada celda muestra la correlación estimada entre dos edades",
       x = "Edad (años)", y = "Edad (años)") +
  theme_minimal() +
  theme(axis.text.x = element_text(angle = 45, hjust = 1))
`} />

**Interpretación de los heatmaps:**
- En **simetría compuesta**, todas las correlaciones fuera de la diagonal son iguales (un solo valor para todos los pares).
- En **AR(1)**, la correlación disminuye progresivamente a medida que nos alejamos de la diagonal (más distancia temporal → menos correlación).
- En **no estructurada**, cada par de edades tiene un valor único de correlación.
- En **exponencial**, el decaimiento es más suave y depende de una función continua.

## 4. Modelos marginales con `gls`

Si no necesitamos los BLUPs de los efectos aleatorios, podemos usar `gls` para modelar directamente la estructura de covarianza. Esto es conceptualmente similar a lo que hacen los modelos marginales (como GEE), pero basado en máxima verosimilitud.

<RCell id="gls-comparacion" title="Modelo marginal con gls: comparación de estructuras" code={`library(nlme)

# gls sin efectos aleatorios, modelando la correlación directamente
gls_cs <- gls(distance ~ age + Sex,
              correlation = corCompSymm(form = ~ 1 | Subject),
              data = Orthodont, method = "ML")
gls_ar1 <- gls(distance ~ age + Sex,
               correlation = corAR1(form = ~ age | Subject),
               data = Orthodont, method = "ML")

# Comparación con lme
mod_ar1 <- lme(distance ~ age + Sex, random = ~ 1 | Subject,
               correlation = corAR1(form = ~ age | Subject),
               data = Orthodont, method = "ML")

# Tabla de coeficientes
resumen <- data.frame(
  Modelo = c("gls AR(1)", "gls AR(1)", "gls AR(1)",
             "lme AR(1)", "lme AR(1)", "lme AR(1)"),
  Parámetro = rep(c("Intercepto", "age", "SexFemale"), 2),
  Estimación = round(c(coef(gls_ar1), fixef(mod_ar1)), 3)
)
print(resumen)

cat("\\nAIC gls AR(1):", round(AIC(gls_ar1), 2),
    "\\nAIC lme AR(1):", round(AIC(mod_ar1), 2))
`} />

<Callout type="tip" title="¿gls o lme?">
  - **`gls`** es un modelo marginal: estima los efectos fijos poblacionales modelando directamente la correlación, sin estimar efectos aleatorios.  
  - **`lme`** es un modelo condicional: incluye efectos aleatorios que inducen correlación.  
  - Si solo te interesa la inferencia poblacional y no necesitás predicciones individuales, `gls` es una alternativa parsimoniosa dentro del ecosistema `nlme`.  
  - **GEE** (con `geepack`) es otra alternativa marginal, pero basada en ecuaciones de estimación en lugar de verosimilitud. En WebR usamos `gls` como aproximación; en tu R local, `geepack` es el estándar.
</Callout>

## 5. Diagnóstico y modelado de heterocedasticidad

¿Qué ocurre si la varianza no es constante? Por ejemplo, si la dispersión de la distancia dental cambia con la edad o difiere entre sexos.

<RCell id="residuos-homocedastico" title="Gráfico de residuos del modelo homocedástico" code={`library(ggplot2)

# Varianza diferente por sexo (varIdent)
mod_var_sex <- lme(distance ~ age + Sex, random = ~ 1 | Subject,
                   correlation = corAR1(form = ~ age | Subject),

                   data = Orthodont, method = "REML")

# Modelo homocedástico para comparar
mod_homo <- lme(distance ~ age + Sex, random = ~ 1 | Subject,
                correlation = corAR1(form = ~ age | Subject),

  Modelo = c("Homocedástico", "VarIdent (Sexo)", "VarPower (edad)"),
  AIC = round(c(AIC(mod_homo), AIC(mod_var_sex), AIC(mod_var_pow)), 2),
  BIC = round(c(BIC(mod_homo), BIC(mod_var_sex), BIC(mod_var_pow)), 2),
  logLik = round(c(logLik(mod_homo), logLik(mod_var_sex), logLik(mod_var_pow)), 2)
)

print(comparacion_var)

mod_var_sex <- lme(distance ~ age + Sex, random = ~ 1 | Subject,

                   weights = varIdent(form = ~ 1 | Sex),
                   data = Orthodont, method = "REML")
mod_var_pow <- lme(distance ~ age + Sex, random = ~ 1 | Subject,

## Flashcard

**Pregunta:** ¿Qué estructura de varianza usarías si notás que la varianza de la respuesta crece sistemáticamente con el tiempo?

**Respuesta:** `varPower` o `varExp` dentro de `lme()` o `gls()`. Ambas permiten modelar la varianza como una función continua de una covariable (como la edad o el tiempo). `varPower` asume que la desviación estándar es proporcional a una potencia de la covariable; `varExp` asume que crece o decrece exponencialmente.

**Pregunta:** En un modelo con `corAR1`, ¿qué significa un valor de φ (phi) cercano a 0.9?

**Respuesta:** φ es el parámetro de autocorrelación de AR(1). Un valor de 0.9 indica que las mediciones adyacentes en el tiempo están muy fuertemente correlacionadas (correlación de 0.9), y esa correlación decae a medida que aumenta la distancia temporal: corr(t, t+2) = 0.9² = 0.81, corr(t, t+3) = 0.9³ ≈ 0.73, etc.

  repeated age / subject=Subject type=un rcorr;
run;

* Exponencial (SP(POW)) *;

  class Subject Sex;
  model distance = age Sex / solution;
  repeated age / subject=Subject type=sp(pow)(age) rcorr;

| Estructura | Función en R (nlme) | Cuándo usarla |
|------------|---------------------|----------------|
| **Simetría compuesta** | `corCompSymm()` | Correlación constante entre cualquier par de mediciones. Simple y robusta. |
| **AR(1)** | `corAR1()` | La correlación decae con la distancia temporal. Ideal para datos longitudinales con tiempo discreto. |
| **No estructurada** | `corSymm()` | Cada par de tiempos tiene su propia correlación. Flexible pero costosa en parámetros. |
| **Exponencial** | `corExp()` | Decaimiento continuo de la correlación con la distancia temporal. Útil con tiempos irregulares. |
| **Varianza constante** | (por defecto) | Asume homocedasticidad. Punto de partida antes de diagnosticar. |
| **varIdent** | `varIdent(form = ~ 1 \| grupo)` | Varianza diferente para cada nivel de un factor (ej. sexo, tratamiento). |
| **varPower** | `varPower(form = ~ covariable)` | La desviación estándar es proporcional a una potencia de la covariable. |

Con estas herramientas, ahora podés elegir y justificar la estructura de covarianza más adecuada para tus datos longitudinales, respaldando cada decisión con criterios numéricos y visualizaciones claras.
